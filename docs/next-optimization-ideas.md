# iRun Workbench 下一轮优化建议

记录日期：2026-07-02  
当前基线版本：v0.3.7

## 可评估优化项

1. 单电站剧情编排器  
   现在 Manila-E 采用按钮切换固定剧情。下一步可以把 `Event Trigger -> Multi-Agent Collaboration -> Disagreement Arbitration -> Closure Report -> Outcomes Flow` 串成一条可暂停、可回放的演示时间线。

2. 真实数据与 P0 剧情分层  
   当前演示数据是 mock 固定剧情。建议后续把剧情文案、指标、事件流拆成独立 JSON，便于替换为真实告警、诊断、工单、排程数据。

3. 地图点位与训练中心坐标校准  
   当前训练中心和部署目标圈层使用静态百分比定位。建议后续建立可视化坐标标定表，避免换背景图后点位偏移。

4. 多语言文案资产化  
   目前中英文文案散落在组件里。建议后续统一到 `i18n` 字典，降低后续改文案和加语言的成本。

5. 无障碍与动效降级  
   本次加强了多处发光和圈层动效。后续可以补 `prefers-reduced-motion` 降级，避免低性能设备或展厅备用机过载。

6. 单电站员工岗位细化  
   本轮已加入剧情机器人头像层。下一步可以继续做可视化点位标定面板，让屋顶、机巢、调度台等空间锚点在后台可调，避免每次手写百分比坐标。

7. 技能市场真实运行态  
   `SKILL MARKETPLACE` 当前偏展示态。后续可增加安装中、失败、权限申请、版本回滚等状态。

8. GitHub Pages / 线上预览自动化  
   如果后续需要频繁演示，建议配置 Pages 或 Actions，把 tag 版本自动发布成稳定预览链接。

9. Manila-E 镜头分镜增强
   当前剧情以叠加层和按钮触发为主。下一轮可加入轻微镜头推拉、局部遮罩聚焦和分镜节奏控制，让每个按钮都像演示脚本的一镜到底。

10. 调度面板状态稳定性
   本轮已降低调度状态行闪动。下一步可把“连接态、输入态、执行态、完成态”拆成明确状态机，减少自动滚动和状态提示互相干扰。

11. 成果流数字资产化
   当前数字增长仍是前端 mock。后续建议把全网闭环、Token、经验回流、损失下降等指标放入独立数据源，便于展厅演示前快速替换。
